# Контекст текущих проблем для нового чата / агента

## Текущее состояние проекта

Мы разрабатываем веб-приложение для анализа графиков акций (stock_analyzer), похожее на TradingView. Стек: Next.js (frontend), FastAPI (backend), lightweight-charts (графики).
Данные получаем от финансового провайдера FMP (Financial Modeling Prep).

## Суть нерешенных проблем

### 1. Рассинхронизация цен и времени на разных таймфреймах

**Симптомы:**

- При переключении таймфреймов (например, с 1H на 4H) цена акции и дата последней свечи отличаются. На 1H может быть цена $1.65, а на 4H — $2.21.
- Последняя свеча отображается точной датой (например, 27 февраля), когда на самом деле уже другое число (март).

**Причина:**
У FMP бесплатного тарифа исторические данные (особенно intra-day) идут с задержкой 15 минут. Чтобы на графике отображалась самая актуальная рилл-тайм цена, мы добавили эндпоинт `/api/v1/charts/quote/{symbol}` (который отдает цену без задержки) и попытались искусственно "склеить" его с массивом исторических свечей на фронтенде в файле `page.tsx` (`app/chart/[symbol]/page.tsx`).

Алгоритм склеивания получился проблемным:

- Математика с `Math.floor(qTime / sec) * sec` (для поиска времени актуальной свечи) конфликтует с часовыми поясами (мы переводим нью-йоркское время FMP в MSK).
- Для дневок (`1day`) формат времени — строковый `'YYYY-MM-DD'`, для остальных — `timestamp`.
- В итоге новая свеча с актуальной ценой либо перезаписывает случайную историческую свечу, либо "улетает" далеко вперед, образуя разрыв, а на разных таймфреймах "склейка" срабатывает по-разному, из-за чего юзер видит разные цены.

**Задача для агента:**
Полностью переписать логику инъекции `quoteData` в массив свечей в `page.tsx`. Либо использовать механизм `update()` самого `lightweight-charts` вместо ручной модификации исходного массива данных, либо перенести логику объединения (склейки) historical + quote на бэкенд (`charts.py`), чтобы фронтенд всегда получал уже готовый, актуальный и правильно сформатированный массив.

### 2. Вертикальная несоосность панелей графиков

**Симптомы:**

- Основной график свечей и нижняя панель индикаторов (например, RSI) не выровнены строго по вертикали (правая шкала цен / оси Y имеет разную ширину).
- Из-за этого вертикальная линия курсора (crosshair) преломляется, а сетка графиков не совпадает.

**Попытки решения:**

- В компоненте `StockChart.tsx` был добавлен параметр `rightPriceScale: { minimumWidth: 70 }`. Однако, возможно, этого значения недостаточно, или ширина динамически съезжает из-за того, что на одной панели значения формата $1,500.00, а на другой 50.00.

**Задача для агента:**
Обеспечить 100% соосность координатной сетки по оси X. Возможно, потребуется использовать API `lightweight-charts` для синхронизации ширины `priceScale` между панелями (`priceScale('right').width()`) или пересмотреть структуру CSS-контейнеров.

## Важные файлы для ознакомления

1. `frontend/app/chart/[symbol]/page.tsx` — здесь живет логика выборки данных (`useQuery`) и сломанная склейка `quoteData` + `data`.
2. `frontend/components/Chart/StockChart.tsx` — здесь инициализируются все инстансы `lightweight-charts`, разделенные по независимым `<div>` панелям. Здесь же логика `minimumWidth` для шкалы.
3. `backend/api/endpoints/charts.py` — бэкенд, преобразующий данные от FMP с конвертацией часовых поясов.
