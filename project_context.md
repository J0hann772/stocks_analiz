# Контекст Проекта и Переменные

Этот файл содержит основную конфигурацию, архитектурные решения и текущий статус разработки для проекта по анализу акций. Весь контекст и комментарии в проекте должны быть на русском языке. Данный файл выступает как **Handoff (точка входа)** для каждого нового чата/сессии.

## Главная миссия проекта

1. **Конструктор стратегий**: Создание пользовательских стратегий с использованием `pandas_ta` или эмуляция индикаторов TradingView.
2. **Скринер акций**: Разовый и автоматический мониторинг американских акций на основе паттернов и критериев индикаторов (по аналогии с Finviz). По дефолту охват S&P 500. Вынесен в фоновую задачу (ARQ queue).
3. **Продвинутые графики**: Инструмент, сопоставимый с TradingView (`lightweight-charts`), для просмотра графиков с кастомными индикаторами (RSI, SMA, EMA), рисунками, таймфреймами и зонами перекупленности/перепроданности.
4. **Мониторинг**: Отслеживание списка акций согласно стратегиям с выводом алертов в Telegram.

## Данные и API

- **Источник**: Financial Modeling Prep (FMP)
- **API Endpoints**: Мы перешли на **Stable API** (`/stable/` вместо `/api/v3/`). Старые эндпоинты возвращают 403 Forbidden. Тикеры передаются как `?symbol=AAPL`, а не в URL-пути.
- **Кэширование FMP**: Настроено в `services/fmp_client.py` (через Redis). Для расчетов (250 свечей) TTL 15 минут, для графиков (500 свечей) TTL 4 часа.
- **Rate Limit FMP**: Реализован `AsyncRateLimiter` (максимум ~12 rps, чтобы не пробить лимит 750 в минуту).

## Технологический стек и Инфраструктура

- **Фронтенд**: Next.js 14, React, TypeScript, `react-query` (автообновление графиков раз в 1 мин), CSS Modules. Библиотека графиков — `lightweight-charts`. Интерфейс разделен на:
  - Авторизация (`/login`).
  - Полноэкранный график (`/chart/[symbol]`).
  - Скринер (`/scanner` - В ПЛАНАХ ИНТЕГРАЦИЯ).
  - Сетка графиков (`/multichart` - В ПЛАНАХ).
- **Бэкенд**: Python 3.12, FastAPI, PostgreSQL (SQLAlchemy с asyncpg), Redis (aioredis).
- **Фоновые Воркеры**: `arq`. Логика скринера перенесена в `background_scan_batch` (`workers/scanner_tasks.py`). Периодический мониторинг избранных акций (Cron) выполняется через `workers/monitor_cron.py`.
- **Docker**: `docker-compose.yml` запускает 4 сервиса: `api` (Backend), `frontend` (Next.js), `redis`, `worker` (ARQ Consumer).

## Что уже реализовано (Текущий Статус)

1. **Базовая структура Бэкенда и БД**: Роутеры `users.py`, `strategies.py`, `scanner.py`, `charts.py` работают. Подключен PostgreSQL.
2. **API Скринера (Асинхронно)**: Роут `/api/v1/scanner/scan` ставит задачу в ARQ и отдает `job_id`. Полный обход S&P500 теперь обрабатывается воркером на фоне.
3. **Графики (UI)**:
   - Исторические данные сливаются на бэкенде с realtime котировками (исправлена проблема рассинхронизации цен). Конвертация внутридневных дат в UNIX Timestamp осуществляется с поправкой на таймзону New York, а на клиенте рендерится в московском времени (UTC+3).
   - Поддерживают панель рисования и "Модальное окно Индикаторов".
   - Умеют строить прямо во фронтенде: SMA, EMA (поверх свечей) и RSI (на отдельной нижней панели с кастомными зонами перекупленности/перепроданности).
   - Жесткая вертикальная синхронизация между главным графиком и индикаторами ($minimumWidth: 95$), синхронизированный crosshair, и прозрачная генерация точек для RSI (whitespace) во избежание разъезжания шкал из-за NaN значений. Используется нативный подход LC через `addBaselineSeries`.
4. **Уведомления**: Написан асинхронный клиент `telegram_signals.py` для отправки сигналов в бот `TG_SIGNAL_BOT_TOKEN`.

## Переменные окружения (.env)

В проекте используются следующие конфиги (ключи не включены в целях безопасности, см. сам файл `.env`):

- `FMP_API_KEY`: Ключ от Financial Modeling Prep API
- `TG_BOT_TOKEN`, `TG_CHAT_ID`, `TG_SIGNAL_BOT_TOKEN`: Телеграм-боты и чаты для уведомлений и скринера
- `TEAM_PASSWORD`: Пароль для регистрации или админских доступов
