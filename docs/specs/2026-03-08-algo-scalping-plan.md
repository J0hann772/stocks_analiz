# План разработки: Алгоритмическая платформа для скальпинга (Стратегия 4H/5m Spring)

Этот документ описывает подробный технический план перевода текущей аналитической платформы на систему алгоритмического скальпинга. План включает в себя используемый стек, ключевые функции и детальные этапы разработки.

## 1. Технологический стек

### Бэкенд

- **Язык**: Python 3.12
- **Фреймворк**: FastAPI (для CRUD и запуска бэктестов)
- **База данных**: PostgreSQL (реляционные данные)
- **ORM / Миграции**: SQLAlchemy 2.0 (асинхронная) + Alembic
- **Кэширование**: Redis (для кэширования "тяжелых" 1-минутных графиков от FMP)
- **Вычисления**: Pandas (для векторизованной агрегации 1m данных в 5m)
- **Источник данных**: Financial Modeling Prep API (FMP) (REST API: `historical-chart/1min`)

### Фронтенд

- **Фреймворк**: Next.js (App Router)
- **Графики**: Lightweight Charts (от TradingView)
- **Стилизация**: TailwindCSS
- **Стейт-менеджемент / API**: React Query (для загрузки истории), Zustand (для стейта часовых поясов и портфеля)

## 2. Ключевые функции продукта (Features List)

### A. Мой портфель (My Portfolio)

- Выезжающая боковая панель с избранными тикерами пользователя.
- Фильтрация тикеров по вкладкам рынков: **Stocks, Crypto, Forex, Gold**.
- Интеграция с функцией **Мульти-графиков** (Multichart) для одновременного просмотра активов из портфеля.

### B. Синхронизация рисунков

- Изменение логики рисования (трендовые линии, SL/TP) — привязка к логическим координатам (`timestamp`, `price`) вместо координат экрана.
- Рисунки автоматически сохраняются на бэкенде.
- Рисунок, сделанный на 4H таймфрейме, мгновенно отображается на 5m графике с правильной привязкой цены ко времени.

### C. Выбор часового пояса (Timezone)

- Возможность менять отображаемое на графиках время (по умолчанию UTC).
- Привязка выбранной зоны к аккаунту пользователя, чтобы настройка не слетала при перезагрузке (сохранение в БД).

### D. Ядро Бэктестера (Backtest Engine)

- **Учитывает 2 типа расписания рынков**:
  - _Акции США_: Торговый день по Нью-Йорку (выходные - пауза). Коридор фиксируется по свече 06:00 - 10:00 NY (30 мин после открытия торгов).
  - _Crypto/Forex/Gold (24/7)_: Торгуют круглосуточно, поиск входов ведется все календарные дни. Коридор фиксируется по свече 00:00 - 04:00 NY.
- Исполнение алгоритма с жестко заданным капиталом (100 акций/лотов) и без шорт-позиций (только Long).
- Эмуляция движения цены внутри свечей на основе **1-минутных (1m) исторических данных** для сверхточного определения первенства срабатывания Стоп-Лосса или Тейк-Профита.
- Выход из сделок либо по SL/TP (соотношение 1:2 R:R), либо **принудительное закрытие в конце суток** (перед формированием новой первой 4H свечи).

### E. Детальная визуализация бэктеста

- Отрисовка маркеров Buy (стрелок) прямо на свечном графике (Lightweight Charts) в точках входа.
- **Отрисовка Коридора**: две белые горизонтальные линии (линии сопротивления и поддержки 4H-свечи), которые тянутся от момента фиксации максимума/минимума свечи и до конца текущих суток.
  - _Важное условие_: Коридор отображается **только на 5-минутном (5m) таймфрейме** и **только в те дни, когда была совершена сделка** (чтобы не перегружать график).
- Подвал (боковая панель): График кривой доходности (Equity), метрики Win Rate и максимальной просадки (Max Drawdown), а также таблица со списком сделок.

## 3. Этапы разработки (Roadmap)

### Этап 1: Подготовка БД и схемы данных

1. Добавить поле `timezone` (default: 'UTC') в модель `User`.
2. Создать модели `Drawing` (для сохранения логических координат рисунков).
3. Создать модель `PortfolioItem` (связка пользователя и избранных активов с типом рынка).
4. Настроить Alembic (если еще не до конца настроен `alembic.ini`) и создать первичные файлы миграции.

### Этап 2: Бэкенд — Инфраструктура и API

1. **FMP Client**: Добавить метод загрузки `historical-chart/1min` (REST).
2. **Redis**: Внедрить кэширование на уровне сервера (загружать год 1-минутных данных очень дорого; кэш необходим для быстрого повторного бэктеста).
3. **CRUD Эндпоинты**:
   - `GET/POST /api/v1/portfolio` — управление портфелем.
   - `GET/POST /api/v1/drawings/{symbol}` — управление рисунками.
   - `PUT /api/v1/users/me/timezone` — выбор часового пояса.

### Этап 3: Бэкенд — Мультирыночное Ядро Бэктестинга (Backtester)

1. **Агрегатор (1m -> 5m, 4H)**: Написать логику сборки старших таймфреймов "на лету" из минутной истории.
2. **Логика расписаний (Schedule Manager)**: Разделить поиск `Коридора` в зависимости от типа тикера (US Equities против 24/7 Asset).
3. **Процессор сигналов (Signal Engine)**: Реализовать логику 4H/5m Spring. Вычисление ложного пробоя, расчет хвоста (<= 50%), выставление SL/TP.
4. **Симулятор исполнения (Execution Simulator)**: Оценка 1m тиков внутри открытой сделки для расчетов PnL. Принудительное закрытие сделок в `23:59 NY` (для 24/7) или в `16:00 NY` (для акций).
5. **Backtest API**: Создать эндпоинт `/api/v1/backtest/{symbol}`, возвращающий готовую кривую эквити и историю сделок (список объектов `{entryTime, exitTime, entryPrice, exitPrice, pnl, corridorHigh, corridorLow}`).

### Этап 4: Фронтенд — Панель навигации (Мой портфель и Таймзона)

1. Добавить селектор `timezone` в футер или хедер графика, синхронизирующийся с бэкендом. Привязать настройку к опции `localization.timeZone` в Lightweight Charts.
2. Добавить выдвижную панель справа (Right Sidebar). Вывести туда элементы `PortfolioItem`.
3. Создать фильтры/вкладки (Tabs: Stocks, Crypto, Forex).

### Этап 5: Фронтенд — Рисование (Drawings)

1. Переработать внутренний флоу рисования: вместо X/Y (пикселей) линии должны инициализироваться как список объектов `{ time: LogicalTime, price: number }`.
2. Настроить загрузку/сохранение рисунков при переключении таймфреймов: например, если переключился с 4H на 5m, React Query обновляет данные, и Canvas перерисовывает линию на тех же ценовых и временных маркерах.

### Этап 6: Фронтенд — Графика Бэктестера (UI)

1. Создать нижнюю панель (Bottom Panel). Добавить выбор дат от/до и кнопку "Запуск Бэктеста".
2. Отобразить кривую `Equity` на отдельном Lightweight Chart внутри нижней панели, а также вывести метрики (Winrate, Total Return).
3. **Маркеры (Buy arrows)**: Интегрировать API Lightweight Charts (`setMarkers`) для отображения входов в лонг прямо над зелеными свечами.
4. **Визуализация Коридора (White Lines)**: При получении результатов бэктеста:
   - Если активен `5m` таймфрейм — отрисовать горизонтальные белые линии от времени первого бара Коридора до времени последнего бара текущих суток (задействовать Price Lines инструментарий или серию `LineSeries`). Отрисовывать линии _только_ в массиве дней прибыльных и убыточных сделок.

### Этап 7: Вылавливание багов и Тестирование (QA)

1. Написать Unit-тесты для бэкенда: проверить, что агрегатор правильно строит свечу 06:00 - 10:00.
2. Подготовить интеграционный тест с синтетическим ложным пробоем на 1-минутных данных. Удостовериться, что Тейк-Профит и Стоп-Лосс не срабатывают в одну и ту же "идеальную" свечу из-за багов аппроксимации.
3. Проверить загрузку кэша: бэктест второго прохождения одного и того же куска истории не должен идти к API FMP.
4. Проверка сдвига свечей на фронте при смене часового пояса.
