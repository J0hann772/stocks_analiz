# Адаптация для алгоритмического скальпинга

Этот список задач описывает шаги по превращению аналитической платформы в систему для алгоритмического скальпинга на базе стратегии 4H/5m Spring.

## 1. Обновление базы данных и моделей

- [ ] Добавить поле `timezone` в модель `User` (по умолчанию 'UTC').
- [ ] Создать модель `Drawing` для хранения синхронизированных рисунков (`symbol`, `type`, `points` (JSON: `[timestamp, price]`)).
- [ ] Создать модель `PortfolioItem` для раздела "Мой портфель" (избранные тикеры).
- [ ] Сгенерировать и применить миграции Alembic.

## 2. Бэкенд API и ядро бэктестинга (FastAPI)

- [ ] **Конвейер данных**: Реализовать загрузчик исторических 1-минутных данных из FMP и логику их агрегации в 1-часовые и 4-часовые таймфреймы.
- [ ] **Ядро бэктестера**:
  - [ ] Инициализировать модель исполнения (100 акций, без шортов).
  - [ ] Реализовать событийно-ориентированную или векторизованную логику, которая проходит по 5-минутным свечам, используя 1-минутные данные внутри них для точного срабатывания SL/TP.
- [ ] **Логика стратегии (4H Spring)**:
  - [ ] Определение контекста: первая 4-часовая свеча основной сессии (09:30 - 13:30 по Нью-Йорку).
  - [ ] Торговое окно: вторая 4-часовая свеча (13:30 - 16:00 по Нью-Йорку).
  - [ ] Условие: 5м свеча закрывается ниже лоу 4H, следующая свеча (или одна из следующих) закрывается выше лоу. Хвост (отклонение) <= 50% ширины 4H канала.
  - [ ] SL: Самая низкая точка хвоста. TP: Риск 1:2 (тейк в 2 раза больше стопа).
  - [ ] Закрытие позиций в конце дня или удержание до срабатывания TP/SL.
- [ ] **Эндпоинты**:
  - [ ] `/api/v1/backtest` для запуска стратегии и получения кривой доходности и списка сделок.
  - [ ] CRUD эндпоинты для `/api/v1/portfolio`.
  - [ ] CRUD эндпоинты для `/api/v1/drawings`.
  - [ ] Обновить `/api/v1/users/me` для поддержки выбора часового пояса.

## 3. Фронтенд UI/UX (Next.js)

- [ ] **Выбор часового пояса**: Добавить селектор таймзоны (стиль TradingView) в футер графика. Синхронизировать с профилем пользователя.
- [ ] **Обновление системы рисования**: Переделать инструменты (трендовые линии, SL/TP) на использование координат `{время, цена}` вместо пикселей экрана. Реализовать сохранение в БД.
- [ ] **Мой портфель**: Добавить выезжающую правую панель со списком избранных тикеров для быстрого переключения графиков.
- [ ] **Мульти-графики**: Интегрировать просмотр нескольких тикеров из портфеля одновременно.
- [ ] **Визуализация бэктеста**:
  - [ ] Отрисовка маркеров Buy (стрелок) на графике `lightweight-charts`.
  - [ ] Добавить разворачиваемую нижнюю панель с графиком Equity и метриками (Win Rate, Max Drawdown).

## 4. Верификация и тесты

- [ ] Написать тесты для логики агрегации 1m -> 5m/4H.
- [ ] Написать интеграционный тест для бэктестера на синтетических данных с паттерном "Spring".
- [ ] Проверить корректность сдвига временной шкалы при смене таймзоны на фронтенде.
